# 🛡️ Project Inbox Defender: Interactive Phishing Awareness Platform

An interactive, multi-level simulator designed to train users on identifying sophisticated phishing attacks and social engineering tactics. This project transitions traditional, passive security training into an engaging, gamified defense grid.

## 🎯 Project Objective
This platform was engineered to satisfy the core objectives of **Task 2: Phishing Awareness Training**:
* **Interactive Module Delivery:** Replacing static presentations with a functional web application.
* **Deception Training:** Teaching users to recognize malicious URLs, spoofed senders, and fake domains.
* **Psychological Analysis:** Educating users on social engineering behavioral hooks (Urgency, Authority, and Scarcity).
* **Real-World Scenarios:** Deploying complex, data-driven attack scenarios modeled after industry incidents.

🔹 Stage 1: Level 0 - The Interactive Briefing
The Interface: Users are met with a cinematic mock terminal.

The Innovation: Utilizing a custom-embedded HTML5/JavaScript camera viewport matrix, users select analytical parameters (Sender, Tone, Link).

The Animation: The application executes a smooth dynamic camera zoom and transform onto the selected code segment, isolating the security indicator visually while rendering an offensive security breakdown log underneath.

🔹 Stage 2: The Perimeter Defense Grid (Levels 1–5)
The platform features a 5-level core engine built on top of a highly structured Pandas pipeline. Each level challenges the user with a singular inbound message vector containing distinct threats:

Level 1 (Beginner): Focuses on basic Typosquatting and explicit time-bound panic cues.

Level 2 (Intermediate): Highlights external tracking subdomains masquerading as consumer utilities.

Level 3 (Advanced): Showcases targeted Spearphishing relying heavily on human authority manipulation.

Level 4 (Expert): Exposes Credential Harvesting mechanics using reverse-proxy single sign-on provider mimicry.

Level 5 (Elite): Explores a sophisticated Supply Chain Dependency Confusion attack explicitly designed to compromise technical development environments.

🔹 Stage 3: The Two-Phase Evaluation Loop
To earn the maximum 5 points per level, users must clear two separate assessment barriers:

Phase 1: Binary Classification (2 Points): The user acts as an incident responder, deciding whether to Quarantine/Report or Trust/Authorize the packet based on visual clues (e.g., hovering over links to view underlying routing paths).

Phase 2: Structural Flaw Identification (3 Points): Users are presented with a multiple-choice radio vector array. They must pinpoint the exact vulnerability vector used by the attacker.

The Intelligence Clue Engine: If stuck, users can deploy an operational clue. Doing so triggers a -1 point penalty deduction from the level's maximum point potential, introducing risk management choices to the gameplay.

🔹 Stage 4: After-Action Diagnostics Dashboard
Upon completing Level 5, the application processes total accumulated points against a telemetry rubric to calculate a precision percentage score and assign an industry-equivalent security rank:

100% Accuracy: Tier 3 Elite Incident Responder

80% - 99% Accuracy: Security Operations Center (SOC) Analyst Tier 1

Under 60% Accuracy: High-Risk User Profile (Remediation Mandated)

Data Layer: Pandas (Scenario processing framework)
